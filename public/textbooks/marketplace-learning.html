Marketplace · Skill · Plugin · Cowork

Marketplace를 “앱 목록”이 아니라 “학습과 작업을 연결하는 조합 시스템”으로 보기

Marketplace를 배우는 목적은 기능 이름을 외우는 것이 아닙니다. 어떤 문제를 어떤 skill, plugin, cowork 조합으로 풀지 판단하는 감각을 만드는 것입니다. 이 교재는 그 판단 흐름을 디자인 중심으로 설명합니다.

10 sections 50 min Product + Workflow Lens
Skill

반복되는 지시와 문맥을 재사용 가능한 레이어로 만든 것입니다.

Plugin / Tool

외부 행동을 실제로 일으키는 실행 지점입니다. 검색, 문서, 파일, 데이터가 여기에 들어옵니다.

01 Prompt Layer 무엇을 시킬지 규정합니다.
02 Skill Layer 반복 문맥을 재사용 가능한 묶음으로 정리합니다.
03 Tool Layer 검색, 문서, 파일, API 같은 외부 행동이 연결됩니다.
04 Workflow Layer 여러 skill과 tool이 한 작업 흐름으로 묶입니다.

Compare

Skill, Plugin, Cowork의 차이

이 셋을 섞어 부르면 설계가 흐려집니다. 목적별로 나눠보세요.

Skill

지식, 톤, 절차, 주의점 같은 문맥을 재사용하는 계층입니다. “어떻게 생각할지”를 고정합니다.

Plugin / Tool

검색, 파일, 문서, API처럼 실제 행동을 일으키는 계층입니다. “무엇을 할지”를 실행합니다.

Cowork

여러 역할 또는 전문 에이전트를 작업 흐름 안에서 협업하게 만드는 계층입니다. “누가 어떻게 나눠서 할지”를 정합니다.

Design Principles

좋은 Marketplace 설계 원칙

이 원칙은 교육 사이트, 팀 도구, 내부 허브에도 그대로 적용됩니다.

하나의 job to be done

하나의 skill은 “무엇이든 도와줌”이 아니라, 한 가지 문제를 빠르게 푸는 쪽이 좋습니다.

문맥과 실행 분리

Skill은 사고 프레임, Plugin은 실행 수단으로 나누면 유지보수가 편해집니다.

권한 최소화

강력한 툴일수록 기본은 닫아두고, 필요한 맥락에서만 열어야 합니다.

학습 가능한 인터페이스

사용자는 “이 조합이 왜 필요한지”를 이해할 수 있어야 다시 씁니다.

Applied Example

교육 사이트에 적용하면

Skill

  • 초보자 눈높이 설명
  • 실습형 답변 형식
  • 실패 복구 우선 답변

Tool

  • 문서 검색
  • 파일 읽기/쓰기
  • 코드 실행 또는 외부 자료 연결
Prompt → Skill → Tool → Workflow

Example:
교육 질문 입력
  → "초보자용 교재 스타일" skill 적용
  → 내부 문서 검색 tool 호출
  → 필요한 경우 파일/예제 생성
  → 결과를 단계형 튜터 UX로 반환
Failure Points

실패가 자주 나는 지점

도구가 많아질수록 “왜 안 되는지”가 흐려집니다. 아래를 먼저 점검하세요.

  • Skill과 Plugin의 책임이 섞여 있음
  • 권한이 너무 넓어서 운영자가 불안해함
  • 사용자가 어떤 조합을 써야 하는지 알 수 없음
  • 좋은 결과 대신 설명만 계속 나와서 체감 가치가 떨어짐

Glossary
