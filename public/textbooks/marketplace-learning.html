Common problems when learning Marketplace

Mixing up the roles of skills and plugins
Thinking more features automatically means better design
Failing to explain why a user should use a specific combination
Core sentence

A good marketplace is not a feature list. It is a structure that clearly shows problem-solving combinations.

Design principle

One skill should make one job-to-be-done clearly better.

Section A

Marketplace role structure

Separating it into three layers reduces confusion.

S
Skill
Reuse context and reasoning frames
P
Plugin / Tool
Execute real external actions
C
Cowork
Design collaboration and division of roles
Skill

A skill packages repeated tone, procedure, guardrails, and explanation style into a reusable layer.

Plugin / Tool

This is the execution layer that performs real actions such as search, docs, file operations, or external APIs.

Cowork

Cowork combines specialized roles into a collaboration structure instead of forcing one flow to do everything.

Section B

Compare skill, plugin, and cowork

Separate what stores context, what executes actions, and what orchestrates collaboration.

Skill

  • Locks in explanation style and rules
  • Strong for repeated use of the same kind of work

Plugin / Tool

  • Executes real external actions
  • Connects search, files, docs, and APIs

Why cowork should be treated separately

Cowork is not just one feature. It is a higher-order arrangement that decides how skills and tools are combined through roles and sequence.

Section C

Good marketplace patterns

Design principles matter more than raw feature count.

One job to be done

A good skill does not do everything. It makes one job clearly better.

Separate context and execution

Maintenance becomes easier when skills hold reasoning frames and tools handle execution.

Minimal permissions

Do not open every tool from the beginning. Attach only what is needed, gradually.

User legibility

Users keep using a combination only when they understand why it exists.

Builder view

A good marketplace is not a feature inventory. It is a designed set of templates that solves user problems.

Operator view

The operator must decide which skills and tools are enabled by default, and how far permissions and cost should go.

Learner view

If learners do not understand why a combination exists, they will not return to it even if it is powerful.
