Marketplace를 단순한 기능 목록이 아니라, Skill과 Plugin과 Cowork가 어떻게 조합되는지 배우는 작업 설계 관점으로 설명합니다. 이 교재도 reference처럼 상단 버튼으로 섹션을 전환하며 보게 만들었습니다.

Marketplace를 배울 때 흔한 문제

Skill과 Plugin의 역할을 섞어서 이해하기
기능 수만 많으면 좋다고 생각하기
사용자가 왜 이 조합을 써야 하는지 설명하지 못하기
핵심 문장

좋은 Marketplace는 기능 목록이 아니라, 문제 해결 조합을 명확히 보여주는 구조입니다.

설계 원칙

하나의 Skill은 하나의 job to be done을 더 잘 해결해야 합니다.

Section A

먼저 세 층으로 나눠야 혼동이 줄어듭니다.

Skill

반복되는 톤, 절차, 주의점, 설명 스타일을 하나의 재사용 레이어로 묶습니다.

Plugin / Tool

검색, 문서, 파일, 외부 API처럼 실제 행동을 일으키는 실행 계층입니다.

Cowork

한 명의 AI가 다 하는 대신, 역할이 다른 전문 흐름을 협업 구조로 묶는 개념입니다.

Section B

Skill, Plugin, Cowork 비교

무엇을 저장하고, 무엇을 실행하고, 무엇을 조합하는지 나눠서 보세요.

Skill

  • 설명 방식과 규칙을 고정
  • 같은 작업을 다시 쓸 때 강함

Plugin / Tool

  • 실제 외부 행동 실행
  • 검색, 파일, 문서, API 연결

Cowork는 왜 따로 봐야 하나?

Cowork는 하나의 기능이 아니라 역할 배치입니다. 어떤 skill과 tool을 어떤 순서와 분업 구조로 조합할지 결정하는 상위 흐름입니다.

Section C

좋은 Marketplace 패턴

기능 수보다 설계 원리가 더 중요합니다.

하나의 job to be done

좋은 skill은 “무엇이든 해줌”이 아니라, 한 가지 문제를 명확하게 더 잘 해결합니다.

문맥과 실행 분리

Skill은 사고 프레임, Tool은 실행 수단으로 나누면 유지보수가 쉬워집니다.

권한 최소화

처음부터 모든 도구를 열지 말고, 필요한 흐름에만 점진적으로 붙입니다.

사용자 이해 가능성

사용자는 왜 이 조합을 써야 하는지 이해해야 계속 사용합니다.

운영자 관점
