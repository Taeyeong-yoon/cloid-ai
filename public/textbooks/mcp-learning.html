MCP를 “API 키”가 아니라 “AI와 도구 사이의 표준 연결”로 이해하기

이 교재는 MCP를 처음 접하는 학습자가 가장 자주 헷갈리는 지점을 먼저 정리합니다. MCP 서버, 클라이언트, 설정 파일, 외부 API 키의 역할을 분리해서 보고, Codex CLI와 Claude Code에서 어떤 식으로 연결 감각이 생기는지 인터랙티브 흐름으로 따라갑니다.

오해 1

API 키만 있으면 MCP가 붙는다고 생각하기 쉽지만, 실제로는 서버와 설정이 먼저입니다.

오해 2

MCP는 하나의 앱이 아니라, AI가 외부 도구를 표준 방식으로 쓰게 하는 연결 규격입니다.

01 Client Claude Code나 다른 AI 클라이언트가 도구를 요청합니다.
02 MCP Server 파일, GitHub, API 같은 기능을 표준 인터페이스로 노출합니다.
03 Config 어떤 서버를 어떤 명령으로 붙일지 등록합니다.
04 Credentials 필요한 경우에만 API 키나 OAuth가 도구 뒤에 붙습니다.
Core Model

MCP의 최소 구조

가장 먼저 이해해야 할 것은 “AI가 곧바로 외부 세계를 만지는 것이 아니라, 중간에 표준화된 서버 층이 있다”는 점입니다.

AI Client

Codex CLI, Claude Code 같은 도구가 사용자 요청을 받아서 도구 사용을 계획합니다.

MCP Server

파일 읽기/쓰기, 이슈 조회, 외부 API 호출 같은 기능을 표준 방식으로 제공합니다.

Config

어떤 MCP 서버를 어떤 명령으로 띄울지, 어디에서 접근할지를 연결합니다.

Credential

GitHub, Drive, 영상 API처럼 외부 서비스가 필요할 때만 키나 토큰이 도구 뒤에 숨어 있습니다.

Compare

API 호출과 MCP 연결의 차이

둘 다 외부 기능을 쓰는 것 같지만, 사용 방식과 학습 포인트가 다릅니다.

API Key만 있을 때

  • HTTP 호출 코드를 직접 써야 합니다.
  • AI는 키를 알아도 그 자체로 도구를 쓸 수 없습니다.
  • 학습자는 “연결 구조”보다 “요청 코드”에 먼저 갇히기 쉽습니다.

MCP를 붙였을 때

  • AI는 도구 목록과 입력 스키마를 보고 사용할 수 있습니다.
  • 도구 실행 방식이 표준화되어 있어 연결 감각이 빨리 잡힙니다.
  • 파일 시스템, GitHub, 커스텀 API를 같은 구조로 다룰 수 있습니다.

교육 관점

  • 처음에는 filesystem MCP처럼 키가 필요 없는 도구가 가장 좋습니다.
  • 그 다음 GitHub나 Google 계열처럼 인증이 필요한 도구로 확장합니다.
  • 영상 생성 API 같은 고비용 도구는 마지막 단계에 붙이는 편이 안전합니다.

Roadmap

초보자를 위한 MCP 학습 순서

아래 순서를 따라가면 “키만 있으면 된다”는 오해를 피하고 구조부터 익힐 수 있습니다.

Decision

API 키는 언제 필요한가

MCP를 공부할 때 이 질문을 제일 많이 합니다. 답은 “항상 필요한 건 아니다”입니다.

키가 필요 없는 경우

  • 로컬 파일 시스템 접근
  • 단순 계산기, 로컬 유틸리티
  • 이미 로컬 자원만 쓰는 도구

키가 필요한 경우

  • GitHub API 호출
  • Google Drive/Docs 접근
  • 영상 생성 API, 지도 API, DB SaaS
AI Client
  └─ calls MCP tool
        └─ MCP server decides how to execute
              ├─ local file tool → no API key
              └─ external SaaS tool → API key or OAuth
Practice

Codex CLI + MCP 실습 흐름 예시

실제 연결은 환경마다 다르지만, 학습 흐름은 이 순서로 잡으면 됩니다.

  1. filesystem MCP를 먼저 붙여서 AI가 현재 작업 폴더를 읽게 합니다.
  2. Codex CLI에서 파일 구조를 요약하게 시킵니다.
  3. 간단한 교육용 결과물 하나를 생성합니다. 예: 수학 문제지, 영어 단어 카드.
  4. 그 다음에만 GitHub나 Google 도구처럼 외부 인증형 MCP를 붙입니다.
Goal: let AI read local content safely

2. Ask the client to summarize folder structure
3. Create one small output
4. Review permissions before adding external APIs
