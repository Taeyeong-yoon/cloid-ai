Section B

MCP를 안 쓸 때 vs 쓸 때

둘 다 비슷해 보여도, 사용자의 체감은 완전히 다릅니다.

MCP 없이

  • 각 API마다 다른 호출 코드를 직접 다뤄야 합니다.
  • AI는 도구 목록을 모른 채 답변만 하게 되기 쉽습니다.
  • 초보자는 구조보다 코드 호출에 먼저 막힙니다.

MCP 사용

  • AI가 도구의 이름과 입력 형식을 보고 사용할 수 있습니다.
  • filesystem, GitHub, 커스텀 API를 같은 연결 감각으로 배울 수 있습니다.
  • 교육에서는 구조 이해가 빨라집니다.
Section C

웹 LLM + MCP vs CLI + MCP

같은 MCP라도 어디서 실행하느냐에 따라 체감이 달라집니다.

웹 LLM + MCP

가볍게 시작하기 좋고 진입 장벽이 낮습니다. 다만 로컬 파일과 시스템 접근은 제한될 수 있습니다.

CLI + MCP

로컬 작업과 자동화에 강합니다. 대신 설정과 환경 이해가 조금 더 필요합니다.

→
MCP 도구 연결 표준
→
CLI 로컬 작업과 자동화

교사 관점

MCP는 학생에게 “AI가 도구를 실제로 쓰는 구조”를 설명하기에 좋은 주제입니다. 처음엔 filesystem MCP처럼 눈으로 확인 가능한 도구가 좋습니다.

개발자 관점

MCP는 여러 외부 도구를 제각각 붙이는 대신, 표준 스키마와 실행 방식으로 정리해 주는 연결 레이어입니다.

운영자 관점

어떤 도구를 열고 어떤 권한을 줄지, 로그와 실패 지점을 어디서 관리할지 결정하기 쉬워집니다.

Section F

첫 실습 실험실: filesystem MCP

가장 쉬운 첫 실습은 키 없는 로컬 도구입니다.

  1. filesystem MCP를 연결한다.
  2. AI에게 현재 폴더 구조를 요약하게 한다.
  3. 작은 결과물 하나를 만든다. 예: 문제지 초안
  4. 그 뒤에만 외부 API형 도구를 붙인다.
Section G

키가 필요 없는 경우

  • 로컬 파일 시스템
  • 간단한 로컬 유틸리티

키가 필요한 경우

  • GitHub API
  • Google Drive/Docs
  • 영상 생성 API
