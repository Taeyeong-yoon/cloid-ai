AI Agent Workflow Patterns

Agent systems are not magic. They are compositions of workflow patterns. This textbook maps planning, tool use, memory, and review into practical structures.

Section A

Four representative patterns

Single loop

The simplest structure where read, generate, and review happen in one loop.

Plan then execute

Build a plan first, then execute step by step.

Tool-calling pattern

A pattern that repeatedly uses external tools and data.

Review gate

Enforce a review step before release to protect quality.

Section B

Single pattern vs composite pattern

Single pattern

  • Simple structure and easy debugging
  • Fits small tasks well

Composite pattern

  • More powerful but harder to design
  • Needs explicit handoff and review rules
Section C

The role of memory

Memory is not just long conversation history. It is a strategy for restoring the state needed for the next step.
