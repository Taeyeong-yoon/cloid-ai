예약 작업 만드는 법

  1. Cowork 탭에서 새 작업 시작 (또는 기존 작업에서)
  2. /schedule 입력
  3. 작업 설명을 상세히 작성 (입력 경로, 출력 형식, 예외 처리 포함)
  4. 주기 선택 (매일, 매주, 평일만, 시간별, 온디맨드)
  5. 확인 → Claude가 자동으로 예약 등록

인기 예약 작업 예시

  1. 일일 브리핑: 매일 아침, Slack/이메일/캘린더를 요약한 브리핑 파일 생성
  2. 주간 보고서: 매주 금요일, Google Drive 데이터를 정리한 주간 요약 생성
  3. 경쟁사 모니터링: 매일, 특정 키워드로 웹 검색 후 변화 기록
  4. 폴더 정리: 매주, 다운로드 폴더를 날짜/유형별로 자동 정리
  5. 회의 준비: 매일 오전, 오늘 캘린더의 미팅별 준비 자료 자동 생성
⚠️ 중요: 예약 작업은 컴퓨터가 켜져 있고 Claude Desktop 앱이 열려 있을 때만 실행됩니다. 앱이 닫혀 있으면 건너뛰었다가 다시 열면 자동 실행합니다.

안전한 사용 가이드

Cowork은 강력한 만큼 안전 수칙을 반드시 알아야 합니다.

🛡️ 파일 삭제 보호 ▼

Cowork은 파일을 영구 삭제하기 전에 반드시 사용자에게 명시적 허가를 요청합니다. "Allow" 버튼을 직접 클릭해야만 삭제가 실행됩니다. 실수로 중요한 파일이 삭제되는 것을 방지하는 안전장치입니다.

📂 폴더 접근 범위 제한 ▼

Cowork은 사용자가 명시적으로 허용한 폴더에만 접근합니다. Documents나 Desktop 전체를 주지 말고, 전용 작업 폴더를 만들어서 그것만 공유하세요. 이것이 보안 경계선입니다.

🔒 프롬프트 인젝션 방어 ▼

Cowork이 인터넷에서 콘텐츠를 읽을 때, 악의적인 지시가 포함되어 있을 수 있습니다(프롬프트 인젝션). Anthropic은 방어 시스템을 구축했지만, 에이전트 안전은 아직 발전 중인 분야입니다. 민감한 작업에서는 결과를 반드시 검토하세요.

💾 데이터 저장 위치 ▼

Cowork 대화 기록은 로컬 컴퓨터에 저장됩니다. Anthropic의 서버 보존 기간 정책 대상이 아닙니다. 감사 로그, 컴플라이언스 API, 데이터 내보내기에도 캡처되지 않습니다. 규제 대상 업무에는 사용하지 마세요.

⚡ 토큰 사용량 주의 ▼

Cowork은 Chat보다 훨씬 많은 토큰을 소모합니다. 복잡한 멀티 스텝 작업은 계산 집약적입니다. 사용량 한도에 자주 걸린다면: 관련 작업을 하나의 세션으로 묶기, 불필요하게 넓은 폴더 공유 피하기, 작업 지시를 구체적으로 작성하여 재시도 줄이기를 시도하세요.

📋 Cowork 핵심 정리

• Chat = 대화형 질의응답
• Cowork = 에이전트형 작업 위임 (목표 → 결과물)
• Claude Code = 개발자용 CLI 에이전트
• Cowork은 로컬 폴더에 직접 접근하여 파일을 읽고/쓰고/만든다
• 서브에이전트가 복잡한 작업을 병렬로 분담한다
• /schedule로 예약 작업을 만들 수 있다 (앱이 열려있어야 실행)
• 플러그인으로 Claude를 직무별 전문가로 만들 수 있다
• 전용 작업 폴더를 만들어 보안 경계를 설정하라

📋 Cowork 첫 시작 체크리스트

□ Claude Desktop 앱 설치 (claude.com/download)
□ Pro/Max/Team/Enterprise 유료 플랜 확인
□ 전용 작업 폴더 생성 (예: ~/cowork-lab/)
□ Cowork 탭 클릭 → 폴더 접근 허용
□ 첫 작업 요청: "이 폴더에 프로젝트 구조를 만들어줘"
□ 결과 파일이 실제로 생성되었는지 확인
□ /schedule로 간단한 예약 작업 테스트
□ 커넥터 연결: Gmail, Calendar 등 토글 ON

📋 예약 작업 프롬프트 템플릿

/schedule

매일 아침 9시에 다음을 실행해줘:

1. 입력: ~/cowork-lab/inbox/ 폴더의 새 파일 확인
2. 처리: 각 파일을 읽고 핵심 내용을 요약
3. 출력: ~/cowork-lab/reports/daily-{날짜}.md 로 저장
4. 형식: 한 줄 요약, 주요 항목 목록, 액션 아이템
5. 예외: 새 파일이 없으면 빈 보고서를 만들지 말 것
