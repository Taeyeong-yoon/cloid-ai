Prompt Engineering
Prompt Engineering in Practice

Great prompting is less about clever phrasing and more about structure design. This textbook shows how to combine role, input context, output format, and evaluation criteria into repeatable practice.

Section A

The four prompt layers

A strong prompt in practice aligns role, input, output, and evaluation.

Role

Sets the point of view and responsibility of the model.

Input

Provides the task, data, and constraints clearly.

Output

Locks the output into a table, bullets, JSON, or a report.

Evaluation

States success criteria in advance to reduce variance.

Section B

Practical patterns

Few-shot

Use 1 to 3 examples to stabilize tone and structure.

Structured output

Define the output schema first if downstream automation matters.

Role framing

Role framing makes expectations and quality standards clearer.

Built-in evaluation

Ask the model to self-check against a checklist after answering.

Section C

Weak prompt vs strong prompt

Weak prompt

  • The goal is vague
  • No output format is specified
  • No success criteria are defined

Strong prompt

  • Role and context are explicit
  • The output format is structured
  • Evaluation criteria are included
